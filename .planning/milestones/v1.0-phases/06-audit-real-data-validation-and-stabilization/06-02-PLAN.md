LEVEL 1 — Eligible chunk count per source:

Run via mcp__postgres__execute_sql:
```sql
SELECT
  s.title,
  s.approval_state,
  s.trusted_state,
  s.status,
  COUNT(v.id) AS total_chunks,
  SUM(CASE WHEN
    (v.metadata->>'approvalState') = 'APPROVED'
    AND (v.metadata->>'trusted') = 'true'
    AND (v.metadata->>'active') = 'true'
  THEN 1 ELSE 0 END) AS eligible_chunks
FROM kb_source s
LEFT JOIN kb_vector_store v ON (v.metadata->>'source_id') = s.id::text
GROUP BY s.id, s.title, s.approval_state, s.trusted_state, s.status
ORDER BY s.created_at;
```

PASS threshold:
- PDF decree (50+ pages): eligible_chunks >= 100
- URL source (single page): eligible_chunks >= 5
- FAIL if eligible_chunks = 0 while total_chunks > 0 (flag propagation bug)

LEVEL 2 — Spot-check content for each source:

```sql
SELECT
  LEFT(content, 300) AS content_preview,
  metadata->>'source_title' AS source_title,
  metadata->>'chunk_index' AS chunk_index
FROM kb_vector_store
WHERE (metadata->>'source_title') ILIKE '%168/2024%'
ORDER BY (metadata->>'chunk_index')::int ASC
LIMIT 5;
```
Repeat with '%100/2019%' and '%Giao thong%' patterns.

Manually compare the content_preview text against the corresponding article text in the actual source document. Verify:
- Text is coherent (not garbled Unicode or encoding errors)
- Article/clause references are intact ("Điều X", "Khoản Y")
- No obvious truncation mid-sentence that loses legal meaning
