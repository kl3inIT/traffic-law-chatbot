## What Was Built

### Batch Ingestion Endpoint

`POST /api/v1/admin/ingestion/batch` accepts a JSON body with a list of `BatchImportItemRequest` items (url, title, sourceType, trustCategory). Each item is processed independently:

1. CRLF-sanitize the URL before logging (T-4.1-02-02 mitigation)
2. Check `kbSourceRepository.findByOriginValue(url)` — if found, return `DUPLICATE` per-item result
3. Otherwise call `ingestionService.submitUrl()` which routes through `SafeUrlFetcher` SSRF boundary (T-4.1-02-04 mitigation)
4. On exception, return `ERROR` per-item result

The method is deliberately NOT `@Transactional` so each item uses its own transaction. The response is always HTTP 200 with a `BatchImportResponse` containing per-item results plus aggregate accepted/rejected counts.

### Source Trust Policy Domain

- **TrustTier enum**: `PRIMARY | SECONDARY | MANUAL_REVIEW`
- **SourceTrustPolicy JPA entity**: maps to `source_trust_policy` table (name, domainPattern, sourceType, trustTier, description, createdAt, updatedAt)
- **Liquibase 007**: creates `source_trust_policy` table and adds `trust_tier` column (default `MANUAL_REVIEW`) to `kb_source`
- **SourceTrustPolicyRepository**: JPA repository with `findByTrustTier(tier)`
- **SourceTrustPolicyService**: CRUD with invariant — never touches ApprovalState/TrustedState/SourceStatus on KbSource
- **TrustPolicyAdminController**: `GET/POST/PUT/DELETE /api/v1/admin/trust-policies`

### Retrieval Gate Invariant (T-4.1-02-01)

Trust tier is purely additive metadata. The retrieval gate (`approvalState == APPROVED && trustedState == TRUSTED && status == ACTIVE`) is unchanged. `SourceTrustPolicyService` contains zero references to these gate fields in production code (Javadoc comment only). Verified by grep.

## Decisions Made

1. **Trust tier is additive only** — SourceTrustPolicyService never modifies ApprovalState, TrustedState, or SourceStatus. This is enforced by code structure and verified by the TrustPolicyServiceTest invariant test.
2. **No @Transactional on batchImport()** — Required so each item runs in its own transaction via IngestionService. A batch-level transaction would cause all items to fail if one throws.
3. **Dedup via findByOriginValue()** — Added to KbSourceRepository. Duplicate detection happens before submitUrl() so SSRF validation is bypassed only for genuinely new URLs.
4. **ROADMAP checkbox** — Already marked `[x]` from prior work; no change needed.
