### 1. Cold Start Smoke Test
expected: Kill any running backend server. Start the application from scratch with `./gradlew bootRun` (or equivalent). Liquibase should run migration 007 (source_trust_policy table + trust_tier column on kb_source) automatically. Server should start without errors and respond to a basic health or ping request.
result: pass

### 2. Batch Ingestion — New URL Accepted
expected: POST /api/v1/admin/ingestion/batch with a body like `{"items":[{"url":"https://example.com/law1","title":"Test","sourceType":"WEBSITE_PAGE","trustCategory":"SECONDARY"}]}` returns HTTP 200 with a per-item result of `ACCEPTED` and `accepted:1, rejected:0`.
result: pass

### 3. Batch Ingestion — Duplicate URL Detected
expected: POST the same URL a second time to /api/v1/admin/ingestion/batch. The per-item result for that URL should be `DUPLICATE` (not an error), and `rejected` count increments. The previously accepted entry is unchanged.
result: pass

### 4. Trust Policy Create
expected: POST /api/v1/admin/trust-policies with `{"name":"Official VN Traffic Laws","domainPattern":"thuvienphapluat.vn","sourceType":"WEBSITE_PAGE","trustTier":"PRIMARY"}` returns HTTP 201 with an `id`, the submitted fields, and `trustTier: "PRIMARY"`.
result: pass

### 5. Trust Policy List
expected: GET /api/v1/admin/trust-policies returns HTTP 200 with a JSON array containing the policy created in Test 4. Response includes `id`, `name`, `domainPattern`, `trustTier`.
result: pass

### 6. Trust Policy Update
expected: PUT /api/v1/admin/trust-policies/{id} with `{"trustTier":"SECONDARY"}` (and other required fields) returns HTTP 200. Subsequent GET shows `trustTier: "SECONDARY"` for that policy.
result: pass

### 7. Trust Policy Delete
expected: DELETE /api/v1/admin/trust-policies/{id} returns HTTP 204 (or 200). Subsequent GET /api/v1/admin/trust-policies no longer contains that policy.
result: pass

### 8. Trust Policy Admin UI — Page Loads
expected: Open the frontend app. The left sidebar should show a "Chính sách tin cậy" navigation item with a shield icon. Clicking it navigates to /trust-policy and shows a two-panel admin page (list on left, CRUD form on right). The list shows existing trust policies or an empty state.
result: pass

### 9. Trust Policy Admin UI — Tier Badges
expected: In the trust policy admin page, policies with tier PRIMARY display a green badge, SECONDARY a yellow badge, and MANUAL_REVIEW a gray badge. Creating a new policy via the form and selecting a tier should show the correct badge color in the list.
result: pass
