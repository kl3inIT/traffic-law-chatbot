<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Batch import request body → IngestionAdminController | User-controlled url, title, sourceType values; each url must pass SSRF check via existing submitUrl |
| Trust policy REST API → SourceTrustPolicyService | Admin-supplied domain_pattern could contain injection payloads if used in dynamic queries |
| Trust tier metadata → retrieval gate | Risk: trust tier could be misread as bypassing approval gate |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-4.1-02-01 | Elevation of Privilege | TrustTier on KbSource — risk that tier bypasses ApprovalState gate | mitigate | SourceTrustPolicyService MUST NOT modify approvalState, trustedState, or status on KbSource. Acceptance criteria verifies zero references to these fields in the service. RetrievalPolicy.RETRIEVAL_FILTER stays hardcoded. |
| T-4.1-02-02 | Tampering | Batch import — CRLF in url/title logged by controller | mitigate | CrlfSanitizer.sanitize() applied to url before logging (pattern established in Plan A). |
| T-4.1-02-03 | Denial of Service | Batch import — oversized batch could overwhelm ingestion executor | accept | No batch size limit in v1. Ingestion executor is bounded (core=4, max=10, queue=100 per application.yaml). Acceptable for admin-only endpoint. |
| T-4.1-02-04 | Spoofing | Batch import — SSRF via malicious url parameter | mitigate | Each item routes through existing IngestionService.submitUrl() which calls SafeUrlFetcher SSRF boundary. No bypass. |
| T-4.1-02-05 | Information Disclosure | TrustPolicyAdminController — error detail in 404 response | mitigate | AppException(TRUST_POLICY_NOT_FOUND) is handled by GlobalExceptionHandler (hardened in Plan A) using prod-masking for generic errors; 404 detail is not sensitive. Accept residual detail in 404 messages as non-sensitive. |
| T-4.1-02-06 | Tampering | domainPattern stored in DB — SQL injection risk if used in LIKE queries | accept | domainPattern is stored as a plain string; all DB access is via JPA named queries (no dynamic JPQL). No SQL injection vector. |
</threat_model>
