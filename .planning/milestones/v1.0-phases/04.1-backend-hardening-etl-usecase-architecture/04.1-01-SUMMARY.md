## What Was Built

### LoggingAspect

`@Aspect @Component` with two advices:
- `@AfterThrowing` — always-on; logs at ERROR with method name. Dev profile logs full stack; non-dev logs only cause class simple name (T-4.1-01-04 info disclosure mitigation).
- `@Around` — dev-profile-gated; logs enter/exit at DEBUG only when `log.isDebugEnabled()`.

**Critical fix discovered:** AspectJ 1.9.25 (bundled with Spring Framework 7) rejects `within(pkg..)` — requires `within(pkg..*)`. The `..` suffix must be followed by `*` to match all types in a package hierarchy.

### CrlfSanitizer

Static utility final class with `sanitize(String)` replacing `\r` and `\n` with `_`. Null-safe (returns `"null"`). Applied to two log call sites in `IngestionAdminController`.

### AsyncConfig Upgrade

Now implements `AsyncConfigurer`. Added `getAsyncUncaughtExceptionHandler()` returning a lambda that logs at ERROR. Added `getAsyncExecutor()` to register an initialized executor as the default so the handler covers all async invocations.

### AppProperties

Typed `@ConfigurationProperties(prefix = "app")` with nested classes:
- `Cors.allowedOrigins` (default: `http://localhost:3000`)
- `Chat.Retrieval.topK` (default: 5)
- `Chat.Grounding.limitedThreshold` (default: 2)
- `Chat.CaseAnalysis.maxClarifications` (default: 2)

Registered via `PropertiesConfig` with `@EnableConfigurationProperties(AppProperties.class)`.

### GlobalExceptionHandler Hardening

Added four new handlers to the existing class:
- **DataAccessException** → HTTP 500; detail masked to `"Failure during data access"` in prod profile.
- **ConstraintViolationException** → HTTP 400 with per-field violations map.
- **NoResourceFoundException** → HTTP 404 with `"Resource not found"`.
- **Generic Exception catch-all** → detail masked to `"Unexpected runtime exception"` in prod profile.

Added `Environment` constructor parameter and `isProdProfile()` helper. Kept no-arg constructor for MockMvc standalone tests.
