<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| ChatService -> VectorStore | Retrieved content from the corpus enters answer generation here |
| Retrieved documents -> CitationMapper | Provenance metadata is transformed into user-visible citations here |
| ChatService -> ChatClient | Prompted model output enters the structured-answer pipeline here |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-05 | Tampering | Retrieved content / prompt context | mitigate | Force all retrieval through `RetrievalPolicy.buildRequest(question, 5)` so only approved, trusted, active chunks enter generation. |
| T-02-06 | Spoofing | CitationMapper | mitigate | Build citations exclusively from `Document` metadata and normalized excerpts; never trust model-generated citation prose. |
| T-02-07 | Information Disclosure | ChatService limited/empty retrieval path | mitigate | `0` documents => `REFUSED`; `1..2` documents => `LIMITED_GROUNDING` with unsupported sections omitted rather than guessed. |
| T-02-08 | Denial of Service | Prompt/model call size | mitigate | Cap retrieval to `topK=5` and excerpts to 280 chars so prompt and response sizes remain bounded. |
</threat_model>
