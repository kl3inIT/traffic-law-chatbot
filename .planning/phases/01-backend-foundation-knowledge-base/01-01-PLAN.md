<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Config -> Database | DB credentials from environment variables, not hardcoded |
| Admin API -> Vector Store | pgvector schema managed by Liquibase, not auto-DDL |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|------------------|
| T-01-01 | Information Disclosure | application.yaml | mitigate | DB_URL, DB_USERNAME, DB_PASSWORD, OPENAI_API_KEY read from env vars with ${VAR:default} — no secrets in committed files |
| T-01-02 | Tampering | Liquibase migrations | mitigate | Migrations use checksums enforced by Liquibase; manual schema changes will be rejected on startup |
| T-01-03 | Elevation of Privilege | pgvector initialize-schema | mitigate | initialize-schema=false prevents Spring AI from silently creating tables; Liquibase owns all DDL |
| T-01-04 | Denial of Service | Async thread pool | accept | ThreadPoolTaskExecutor bounded to max-size=10 and queue-capacity=100; overflow rejected with task rejected exception, logged and surfaced as job failure |
</threat_model>
