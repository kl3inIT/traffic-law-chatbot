    - @Component @RequiredArgsConstructor @Slf4j @Transactional

    - Pipeline steps executed in order, each updating job.stepName and saving:

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Admin API -> UrlPageParser | Admin-submitted URL must be validated before HTTP fetch is performed |
| IngestionOrchestrator -> VectorStore | Chunks written to vector store must always carry trusted=false and active=false until approval |
| File upload -> Tika | Uploaded bytes processed by Tika; malicious files must not execute code |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|------------------|
| T-03-01 | Server-Side Request Forgery | UrlPageParser.fetchAndParse() | mitigate | Validate parsed host against InetAddress.isLoopbackAddress(), isLinkLocalAddress(), isSiteLocalAddress() before HTTP fetch. Throw AppException(URL_NOT_ALLOWED) for private addresses. Only http/https schemes accepted. |
