## Gaps

### G1 — StructuredOutputValidationAdvisor rejects extra LLM fields (blocker)

LLM (Claude Sonnet via OpenRouter) returns extra top-level fields `confidence`, `intent`, `note` that are not declared in `LegalAnswerDraft`. Schema generated by `BeanOutputConverter` uses `additionalProperties: false`, so validation fails on every response:

```
property 'confidence' is not defined in the schema and the schema does not allow additional properties
property 'intent' is not defined ...
property 'note' is not defined ...
```

With `maxRepeatAttempts=1` (D-10, Pitfall 6), the retry also fails and the chain falls through to the grounding refusal.

Evidence: `build/test-results/liveTest/TEST-com.vn.traffic.chatbot.chat.regression.VietnameseRegressionIT.xml` (live run 2026-04-18T21:58).

Likely origin: a DB-stored `systemPrompt` in `ai_parameter_set` that instructs the LLM to emit `confidence`/`intent`/`note`. `LegalQueryAugmenter.java:50` loads it via `paramProvider.getString("systemPrompt", ...)`.

Candidate fixes (pick one):
- A — prompt-side: update `LegalQueryAugmenter.buildPromptTemplate` (~line 69) to explicitly forbid extra fields ("Khong them confidence/intent/note hoac bat ky truong nao khac").
- B — schema-side: add `confidence`/`intent`/`note` as optional fields to `LegalAnswerDraft.java`. Risk: may break ARCH-05 byte-for-byte parity with Phase 8.
- C — source-side: fix the DB systemPrompt so it no longer asks for those fields.

### G2 — Array-typed schema fields returned as strings

Schema declares `List<String>` for `legalBasis`, `penalties`, `requiredDocuments`, `procedureSteps`, `nextSteps`. LLM returns plain strings for them, producing:

```
/legalBasis: string found, array expected
/penalties, /requiredDocuments, /procedureSteps, /nextSteps: same
```

Fix: strengthen `buildPromptTemplate` (`LegalQueryAugmenter.java:69`) to state that these fields MUST be JSON arrays of strings (e.g. `"legalBasis": ["...", "..."]`). Consider a one-shot example in the template.

### G3 — maxRepeatAttempts=1 too low under current conditions

With G1+G2 making every first attempt invalid, one retry is not enough. Once G1/G2 are fixed, current value of 1 is likely fine (T-9-04 monitor retry rate before tightening to 0).
