<objective>
Close Phase 9 UAT gaps G1, G2, G3 so the 20-query Vietnamese regression passes live and Phase 9 can be marked complete.

Root cause (from 09-HUMAN-UAT.md): `StructuredOutputValidationAdvisor` rejects every LLM response because (G1) the model emits extra top-level fields `confidence`, `intent`, `note` that `BeanOutputConverter` schema forbids (`additionalProperties:false`), and (G2) list-typed fields `legalBasis`, `penalties`, `requiredDocuments`, `procedureSteps`, `nextSteps` are returned as bare strings instead of JSON arrays. With `maxRepeatAttempts=1` (D-10) the single retry also fails and the chain falls through to `AnswerComposer.composeRefusal()` — explaining the 0/20 live pass rate.

Purpose: Per UAT constraint, prefer the prompt-side fix (Candidate A) over widening the DTO schema (Candidate B) because ARCH-05 requires byte-for-byte `ChatAnswerResponse` JSON parity with Phase 8. Also audit the DB-stored `systemPrompt` (Candidate C) which is loaded at runtime by `LegalQueryAugmenter.java:50` — it is the likely origin of the bad instructions.

Output:
  - Tightened `LegalQueryAugmenter.buildPromptTemplate` that (a) explicitly forbids extra top-level JSON fields, (b) requires JSON arrays for the 5 list-typed fields, (c) includes a one-shot example showing the exact shape.
  - DB audit checkpoint: inspect and, if necessary, sanitize `ai_parameter_set.system_prompt` for active rows.
  - Live-test gate: re-run the three live integration tests with `OPENROUTER_API_KEY` set and confirm closure of UAT items 1-5.

```java
    List<String> legalBasis,         // MUST be JSON array
```

These are the ONLY 8 permitted top-level JSON keys. `BeanOutputConverter` emits a schema with `additionalProperties:false`. The LLM MUST NOT return `confidence`, `intent`, `note`, or any other top-level key.

```java
static PromptTemplate buildPromptTemplate(String systemPrompt) {
    String template = systemPrompt + "\n"
        + "Chỉ bao gồm các mục liên quan theo danh sách: " + ... + ".\n"
        + "Các phần nội dung được phép điền là: " + ... + ".\n"
        + LABEL_RULE + "\n"
        + "Chỉ trả về duy nhất một object JSON hợp lệ, không dùng markdown, không thêm giải thích.\n"
```
