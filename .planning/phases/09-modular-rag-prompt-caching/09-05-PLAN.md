# Plan 09-05 — IntentClassifier Dedicated ChatClient Wiring (G5 close-out)

## Context

Plan 09-04 Task 2 investigated G5 and CONFIRMED root cause H1: every entry in `chatClientMap` carries `validationAdvisor` bound to `LegalAnswerDraft.class`. When `IntentClassifier.classify(...).entity(IntentDecision.class)` runs through the same chain, the validation advisor coerces the LLM response through the LegalAnswerDraft schema → `BeanOutputConverter` parse failure → caught at `IntentClassifier.java:52` → silent fallback to `IntentDecision(LEGAL, 0.0)` per D-02.

Scope exceeded the 09-04 Task 2 budget (≤1 file, ≤30 LOC). This plan closes G5.

## Tasks

### Task 1 — Add sibling `intentChatClientMap` bean (~15–20 LOC)

**File:** `ChatClientConfig.java`

Extract the OpenAiChatModel build loop into a helper or iterate a second time to produce a parallel map:

```java
@Bean("intentChatClientMap")
public Map<String, ChatClient> intentChatClientMap(AiModelProperties modelProperties) {
    // same OpenAiApi + OpenAiChatOptions + OpenAiChatModel build as chatClientMap
    // but the ChatClient is built WITHOUT .defaultAdvisors(...) — plain client only
    Map<String, ChatClient> map = new LinkedHashMap<>();
    for (var entry : modelProperties.models()) {
        OpenAiChatModel chatModel = buildChatModel(entry, modelProperties);  // shared helper
        map.put(entry.id(), ChatClient.builder(chatModel).build());
    }
    return map;
}
```

Refactor the existing `chatClientMap` bean to call the same `buildChatModel(...)` helper to avoid duplicating OpenAiApi / RestClient / retry / observation wiring.

### Task 2 — Qualify `IntentClassifier` injection (~2 LOC)

**File:** `IntentClassifier.java:32`

```java
private final @Qualifier("intentChatClientMap") Map<String, ChatClient> chatClientMap;
```

(Lombok `@RequiredArgsConstructor` picks up the field qualifier.)
