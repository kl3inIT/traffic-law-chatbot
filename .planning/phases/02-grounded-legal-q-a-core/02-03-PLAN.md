<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| public client -> PublicChatController | External users submit untrusted JSON here |
| controller -> ChatService | Validated user input enters the grounded answer pipeline here |
| Spring context -> ChatClient / ChatModel | Provider-backed model execution is activated here |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-09 | Tampering | PublicChatController request body | mitigate | Use `@Valid @RequestBody ChatQuestionRequest` and rely on `GlobalExceptionHandler` to return typed 400 ProblemDetail responses for invalid inputs. |
| T-02-10 | Elevation of Privilege | Route namespace | mitigate | Keep public chat on `/api/v1/chat` and do not reuse `/api/v1/admin/*` constants or controller packages. |
| T-02-11 | Denial of Service | Integration/runtime configuration | mitigate | Keep retrieval bounded by `app.chat.retrieval.top-k: 5` and mock external model calls in tests so no live dependency blocks CI. |
| T-02-12 | Repudiation | API behavior drift | mitigate | Add controller and end-to-end integration tests that pin the response envelope, refusal behavior, and validation errors. |
</threat_model>
