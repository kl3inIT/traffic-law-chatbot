<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Admin URL input -> SafeUrlFetcher | Untrusted URL crosses into the network-fetch boundary. |
| SafeUrlFetcher -> IngestionOrchestrator | Redirect and response metadata cross into persistence and parsing flow. |
| IngestionOrchestrator -> fetch snapshot persistence | Provenance fields become audit data used later by admin and retrieval workflows. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01.1-01 | S | `SafeUrlFetcher` | mitigate | Validate scheme and resolved host before any request; reject loopback, link-local, site-local, and malformed hosts per D-08. |
| T-01.1-02 | T | `KbSourceFetchSnapshot` persistence | mitigate | Persist `requestedUrl` and `finalUrl` only from the fetch contract, not from ad hoc fallback logic in the orchestrator. |
| T-01.1-03 | R | URL ingestion provenance | mitigate | Add regression tests that prove redirect cases store distinct requested/final URLs so audit trails remain trustworthy. |
| T-01.1-04 | I | redirect handling | accept | Final public redirect targets are stored as provenance because this system already records source origins; no additional secret material is introduced. |
| T-01.1-05 | D | Jsoup fetch path | mitigate | Keep request timeout bounded and fail the job cleanly through existing async error handling. |
| T-01.1-06 | E | parser boundary creep | mitigate | Keep Spring AI types out of the fetch contract and preserve `ParsedDocument` as the domain boundary per D-02. |
</threat_model>
