<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| SafeUrlFetcher -> SpringAiHtmlParser | Already-fetched untrusted HTML crosses into commodity parsing internals. |
| SpringAiHtmlParser -> ParsedDocument | External library output is translated into repo-owned domain records. |
| IngestionOrchestrator -> VectorStore | URL-imported chunks become retrievable only if metadata remains approval/trust compatible. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01.1-07 | T | `SpringAiHtmlParser` mapping | mitigate | Map Spring AI output explicitly into `ParsedDocument`; do not pass library `Document` objects across the domain boundary. |
| T-01.1-08 | T | vector metadata | mitigate | Leave `TextChunker` and `buildMetadata()` compatibility keys intact so retrieval and inspection contracts do not drift. |
| T-01.1-09 | D | HTML reader processing | mitigate | Keep bounded fetch size/time behavior in the custom fetcher and fail jobs through existing async failure handling. |
| T-01.1-10 | I | parser provenance | mitigate | Persist parser name/version on `KbSourceVersion` for URL HTML imports so admins can audit which parser path produced the indexed content. |
| T-01.1-11 | E | trust-boundary erosion | mitigate | Keep SSRF validation and redirect capture in the custom fetcher; Spring AI only parses already-fetched content per D-08 and D-09. |
</threat_model>
