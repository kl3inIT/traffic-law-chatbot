## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User query → CachingEmbeddingModel.normalize → SHA-256 key | Untrusted text crosses into cache-key computation. |
| EmbeddingModelChangedEvent publisher → EmbeddingCacheInvalidator | ApplicationContext-internal; any bean can publish. |

## STRIDE Threat Register (ASVS L1)

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-04 | T (Tampering) — Cache poisoning via key collision | CachingEmbeddingModel.call single-instruction path | mitigate | SHA-256 is collision-resistant for 64-byte digest; key prefixed with `modelId` so cross-model poisoning impossible (D-15). Risk: zero practical within 10_000-entry bounded cache. |
| T-07-05 | I (Information Disclosure) — Stale vector after admin swaps embedding model | CachingEmbeddingModel returns cached dim-mismatched float[] | mitigate | Dimension-mismatch guard (D-16): stored as `CachedVector(float[], int)`; consumer asserts dim match, evicts otherwise. Belt-and-braces: `EmbeddingModelChangedEvent` → `cache.clear()` (D-17) — now auto-gated by `EmbeddingCacheContextLoadTest`. |
| T-07-06 | D (Denial of Service) — Unbounded cache growth | CaffeineCacheManager | mitigate | `maximumSize(10_000)` hard cap + TTL safety ceiling (D-18). Caffeine evicts via W-TinyLFU when cap reached. |
| T-07-07 | E (Elevation of Privilege) — Any bean publishing `EmbeddingModelChangedEvent` clears cache | EmbeddingCacheInvalidator | accept | Effect is strictly a cache flush (no data modification); worst case is a latency dip as cache refills. No trust-boundary escalation. |
| T-07-08 | S (Spoofing) — Cached entry attributed to wrong model | CachingEmbeddingModel cache key | mitigate | Key includes `modelId` prefix; different model → different key namespace, never conflated. |
