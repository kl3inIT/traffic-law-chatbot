    @Component
    @RequiredArgsConstructor
    public class ActiveParameterSetProvider {
        private final AiParameterSetRepository repository;
        private final ObjectMapper yamlMapper; // initialized as new ObjectMapper(new YAMLFactory()) in a @Bean or inline

        /**
         * Returns the parsed YAML content of the currently active parameter set.
         * Falls back to empty map if no active set exists.
         * NOT cached -- reads fresh from DB on every call.
         * Caching can be added later if needed.

        public Map<String, Object> getActiveParams() {
            return repository.findByActiveTrue()
                    .map(ps -> parseYaml(ps.getContent()))
                    .orElse(Map.of());
        }

        // Typed accessors for common paths:

        public double getDouble(String dotPath, double fallback) { ... }
        public List<Map<String, Object>> getList(String dotPath) { ... }

        // dotPath navigation: "messages.disclaimer" -> params.get("messages").get("disclaimer")
        // Use nested Map traversal. Return fallback if any level is null.
    }

## Trust Boundaries

|----------|-------------|
| Parameter set YAML -> Runtime behavior | Admin-controlled YAML content affects AI prompt, retrieval, and clarification logic |
| Browser CORS -> Spring | Cross-origin requests from frontend to backend |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-05 | Tampering | ActiveParameterSetProvider | mitigate | YAML is parsed as data (Map<String, Object>), never executed as code; fallback values ensure degraded operation if YAML is malformed |
| T-04-06 | Tampering | CORS config | mitigate | allowedOrigins is configurable via property, defaults to localhost:3000 only; production deployment must set explicit origin |
| T-04-07 | Denial of Service | ChatPromptFactory | accept | Malformed systemPrompt could produce bad AI output but does not crash the system; admin responsibility |
</threat_model>
