## Spacing Scale

Declared values (multiples of 4). Uses Tailwind default scale which maps 1 unit = 4px.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`gap-1`, `p-1`) | Icon gaps, inline badge padding |
| sm | 8px (`gap-2`, `p-2`) | Compact element spacing, button icon gap |
| md | 16px (`gap-4`, `p-4`) | Default element and card padding |
| lg | 24px (`gap-6`, `p-6`) | Section padding, dialog padding |
| xl | 32px (`gap-8`, `p-8`) | Layout column gaps |
| 2xl | 48px (`gap-12`) | Major section breaks |
| 3xl | 64px (`gap-16`) | Page-level vertical rhythm |

Exceptions:
- Chat input area minimum height: 44px (touch target floor, per accessibility standard)
- Sidebar width: 256px collapsed nav rail / 280px expanded (shadcn Sidebar default)
- Thread list panel width: 280px fixed (mirrors sidebar width, does not flex)
- DataTable row height: 48px minimum

## Typography

Uses Inter font loaded via `next/font/google`. All sizes in px; Tailwind class equivalents shown.

| Role | Size | Tailwind | Weight | Line Height | Usage |
|------|------|----------|--------|-------------|-------|
| Body | 14px | `text-sm` | 400 (regular) | 1.5 | Chat bubbles, table cell content, form field values |
| Label | 12px | `text-xs` | 500 (medium) | 1.4 | Badges, table headers, sidebar nav items, timestamp |
| Heading | 20px | `text-xl` | 600 (semibold) | 1.2 | Page titles, dialog titles, section headings |
| Display | 28px | `text-3xl` | 700 (bold) | 1.1 | Empty state headings only |

**Weight constraint:** Only weight 400 and 600 are used in the main UI. Weight 500 is permitted only for labels/badges. Weight 700 is reserved for empty state display headings only.

**Source:** Default sensible values aligned with shadcn/ui defaults; no override in upstream artifacts.

## Color

shadcn/ui CSS variable system. All values reference CSS custom properties — do not hardcode hex in components.

| Role | CSS Variable | Tailwind Class | Usage |
|------|-------------|----------------|-------|
| Dominant (60%) | `--background` | `bg-background` | Main content area, chat thread background |
| Secondary (30%) | `--card` / `--sidebar` | `bg-card`, `bg-sidebar` | Sidebar shell, cards, DataTable rows |
| Accent (10%) | `--primary` | `bg-primary`, `text-primary` | See reserved list below |
| Destructive | `--destructive` | `bg-destructive`, `text-destructive` | Destructive actions only — see below |
| Muted | `--muted` | `bg-muted`, `text-muted-foreground` | Placeholder text, disabled states, timestamps |
| Border | `--border` | `border` | Card borders, dividers, input outlines |

**Accent (`--primary`) reserved for exactly these elements:**
1. "Gửi tin nhắn" (Send) button in chat input
2. "+ Cuộc hội thoại mới" (New Thread) button in sidebar
3. Active thread highlight in thread list
4. Active nav item in sidebar
5. "Kích hoạt" (Activate) button for AI parameter sets
6. "Tạo mới" (Create) button in admin screens

**Destructive (`--destructive`) reserved for exactly these elements:**
1. "Từ chối" (Reject) source action button
2. "Xóa" (Delete) parameter set button
3. Confirmation dialog destructive confirm button

**Source:** shadcn default palette. 60/30/10 split applied per template standard. Accent list derived from D-05, D-06, D-07, D-10 (CONTEXT.md).

## Interaction Contracts

### Chat — Thread List (D-06, D-09)
- Thread list renders in the sidebar under the "Chat" nav section
- Each thread row shows: thread title (derived from first user message, max 40 chars, truncated with ellipsis) + relative timestamp
- Active thread row has `bg-primary/10` background with left 2px `bg-primary` border
- "+ Cuộc hội thoại mới" button is pinned at the top of the thread list, uses `variant="default"` (primary color)
- Clicking a thread navigates to `/threads/[id]` — no page reload (App Router client navigation)
- Thread list is scrollable via `ScrollArea`; new threads prepend to the list (no pagination in Phase 4)

### Chat — Message Bubbles (D-07, D-08)
- User messages: right-aligned, `bg-primary text-primary-foreground`, rounded `rounded-lg`, max width 70% of thread area
- AI messages: left-aligned, `bg-card border`, rounded `rounded-lg`, max width 85% of thread area
- Clarifying question AI messages: same styling as regular AI bubbles — no special treatment (D-08)
- Regular Q&A answer: rendered as plain markdown text inside the AI bubble
- Scenario answer: rendered as shadcn `Accordion` with 5 sections inside the AI bubble:
  - "Sự kiện được xác định" (Facts)
  - "Quy tắc áp dụng" (Applicable Rule)
  - "Hậu quả có thể" (Likely Consequence)
  - "Hành động được khuyến nghị" (Recommended Actions)
  - "Nguồn tài liệu" (Sources)
  - All sections default to collapsed; "Nguồn tài liệu" section is always expanded by default
- Citation sources render as `Badge` components inside the Sources section
- Disclaimer text renders in `text-muted-foreground text-xs` below the answer content
- AI typing / loading state: three animated `Skeleton` lines inside an AI bubble shell

### Chat — Input Area
- Sticky at the bottom of the thread view
- `Textarea` with auto-resize up to 5 rows then scrolls
- "Gửi" (Send) button to the right, `variant="default"`, disabled when input is empty or when AI is responding
- Press Enter to send; Shift+Enter for newline
- Minimum touch target for Send button: 44px × 44px

### Admin — Source Management DataTable (D-10, ADMIN-01)
- Columns: Tiêu đề / URL, Loại nguồn, Trạng thái (badge), Ngày tạo, Hành động
- Status badges: PENDING = `bg-yellow-100 text-yellow-800`, APPROVED = `bg-blue-100 text-blue-800`, ACTIVE = `bg-green-100 text-green-800`, REJECTED = `bg-red-100 text-red-800`
- Action column uses `DropdownMenu` — shows only actions valid for current status:
  - PENDING: Phê duyệt, Từ chối
  - APPROVED: Kích hoạt, Từ chối, Nhập lại
  - ACTIVE: Hủy kích hoạt, Nhập lại
  - REJECTED: Nhập lại
- Destructive actions (Từ chối, Hủy kích hoạt) trigger `AlertDialog` confirmation before calling API
- Table supports column sorting; no pagination in Phase 4 (full list load via TanStack Query)

### Admin — Knowledge Index (ADMIN-02)
- Read-only screen with two `Card` components:
  - "Tóm tắt chỉ mục" (Index Summary): total chunks, active/inactive counts
  - "Sẵn sàng truy xuất" (Retrieval Readiness): approved count, trusted count, active count, combined eligible count
- Data from `/api/v1/admin/chunks/readiness` and `/api/v1/admin/index/summary`
- Refresh button (icon only, with `Tooltip`) to re-fetch; uses TanStack Query `refetch`
- No editing controls — this screen is observation-only

### Admin — AI Parameter Sets (ADMIN-03, D-10)
- List view: `DataTable` with columns: Tên, Trạng thái (Active badge), Ngày tạo, Hành động
- Active row shows `Badge variant="default"` ("Đang hoạt động") on the Tên column
- Action column per row: Chỉnh sửa (Edit), Sao chép (Copy), Kích hoạt (Activate — primary), Xóa (Delete — destructive)
- Delete triggers `AlertDialog` confirmation
- Create/Edit opens a `Dialog`:
  - Field: Tên bộ tham số (name) — `Input`
  - Field: Nội dung YAML (content) — `Textarea` with monospace font (`font-mono text-sm`), min-height 320px
  - Copy action pre-fills the Dialog with the source set's name suffixed " (bản sao)" and its YAML content
- One active set at a time: Activating one set deactivates all others (backend enforces; UI reflects immediately via TanStack Query cache invalidation)

## Loading and Error States

| State | Treatment |
|-------|-----------|
| Thread list loading | 5× `Skeleton` rows at 48px height each |
| Chat message loading | AI bubble shell with 3× `Skeleton` lines |
| DataTable loading | 8× `Skeleton` rows at 48px height, matching column count |
| Card/summary loading | `Skeleton` block matching card dimensions |
| API error (non-destructive) | `Alert variant="destructive"` inline above affected content |
| API error (action failure) | `Alert` inside the Dialog or below the triggering button |
| Network offline | `Alert` at page top: "Không thể kết nối đến máy chủ. Vui lòng kiểm tra kết nối." |

TanStack Query retry: 1 automatic retry on network error, no retry on 4xx responses.
